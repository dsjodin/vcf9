# VCF9_deploy - Codebase Documentation

VMware Cloud Foundation 9 automated deployment toolkit for nested lab environments.
Deploys a full VCF 9.0.1 stack: management domain, workload domain, NSX edge clusters,
and vSphere with Tanzu (Supervisor Cluster) -- all on nested ESXi running inside a
physical vCenter.

Original scripts by William Lam (williamlam.com), extended with workload domain
automation, edge cluster deployment, supervisor cluster enablement, and unified
DNS/config generation.

---

## Architecture Overview

```
Physical vCenter (labvcsa.lab.io)
  |
  +-- vApp: Nested-VCF-9-Lab-<id>
       |
       +-- VCF Installer VM (vcfinst) -----> VCF Installer API
       |
       +-- Management Domain ESXi (esx01-04)
       |     |
       |     +-- SDDC Manager (sddcm01) ---> SDDC Manager API
       |     +-- vCenter (mgmt-vc)
       |     +-- NSX Manager (nsx01/nsx01a)
       |     +-- VCF Operations (ops)
       |     +-- VCF Automation (auto01)
       |     +-- Edge Nodes (m01edge01, m01edge02)
       |           |
       |           +-- Tier-0 (m01-t0-gw01, ASN 65001)
       |                 |
       |                 +-- BGP --> VyOS Router (ASN 65000)
       |
       +-- Workload Domain ESXi (esx05-07)
             |
             +-- vCenter (wld-vc)
             +-- NSX Manager (nsx02/nsx02a)
             +-- Edge Nodes (wldedge01, wldedge02)
             |     |
             |     +-- Tier-0 (w01-t0-gw01, ASN 65002)
             |           |
             |           +-- BGP --> VyOS Router (ASN 65000)
             |
             +-- Supervisor Cluster (Kubernetes/Tanzu)
                   +-- Pod CIDR: 10.244.0.0/20
                   +-- Service CIDR: 10.96.0.0/24
                   +-- Ingress: 10.2.220.0/24
                   +-- Egress: 10.2.221.0/24
```

## Deployment Workflow

Execute scripts in this order:

```
Step 1: CreateSharedVCF-VVF-DNS-Zones.ps1    (prep: DNS zones + config files)
Step 2: vcf-automated-fleet-deployment.ps1    (deploy MGMT domain + ESXi VMs)
Step 3: vcf-edge-cluster-deployment.ps1       (deploy NSX edge clusters)
Step 4: vcf-automated-wld-deployment.ps1      (deploy workload domain)
Step 5: vcf-supervisor-cluster-deployment.ps1  (deploy Kubernetes supervisor)
```

Each script takes `-EnvConfigFile <path>` to load the config (dot-sourced .ps1).

## Configuration Reference

All deployment scripts accept `-EnvConfigFile` pointing to a dot-sourced .ps1 file.
The primary config is `SharedConfigs/VCFDeploymentConfig.ps1`.

### Key Variable Groups

**Physical Environment:**
- `$VIServer` - Physical vCenter (labvcsa.lab.io)
- `$VMCluster`, `$VMDatastore`, `$VMNetwork` - Target cluster/storage/network
- `$NestedESXiApplianceOVA`, `$VCFInstallerOVA` - OVA image paths

**VCF Installer:**
- `$VCFInstallerFQDN`, `$VCFInstallerIP` - Installer VM address
- `$VCFInstallerProductVersion` - VCF version (9.0.1.0)
- `$VCFInstallerSoftwareDepot` - "online" or "offline"

**SDDC Manager:**
- `$SddcManagerHostname`, `$SddcManagerIP` - SDDC Manager address

**Nested ESXi Hosts:**
- `$NestedESXiHostnameToIPsForManagementDomain` - Hashtable {hostname = IP}
- `$NestedESXiHostnameToIPsForWorkloadDomain` - Hashtable {hostname = IP}
- `$NestedESXiMGMTvCPU/vMEM/CapacityvDisk/BootDisk` - MGMT resources
- `$NestedESXiWLDvCPU/vMEM/CapacityvDisk/BootDisk` - WLD resources

**Network CIDRs:**
- `$NestedMGMTESXi*NetworkCidr` - MGMT domain network CIDRs
- `$NestedWLDESXi*NetworkCidr` - WLD domain network CIDRs

**Edge Clusters:**
- `$MGMTEdge*` - Management edge cluster config (nodes, TEP, uplinks, BGP)
- `$WLDEdge*` - Workload edge cluster config

**Supervisor:**
- `$Supervisor*` - Supervisor cluster sizing, CIDRs, storage, content library

## Resource Requirements

### Per Nested ESXi VM

| Domain | vCPU | RAM | Boot Disk | Capacity Disk |
|--------|------|-----|-----------|---------------|
| MGMT | 24 | 160 GB | 64 GB | 500 GB |
| WLD | 24 | 128 GB | 64 GB | 500 GB |

### Total Lab Footprint (minimum)

- 4 MGMT hosts: 96 vCPU, 640 GB RAM, 2.3 TB storage
- 3 WLD hosts: 72 vCPU, 384 GB RAM, 1.7 TB storage
- VCF Installer + SDDC Manager + vCenter + NSX: additional overhead
- **Estimated total: ~200 vCPU, ~1 TB RAM, ~5 TB storage**

## Prerequisites

- PowerShell Core 7.x
- VMware PowerCLI module (`VMware.PowerCLI`)
- VMware.Sdk.Vcf.SddcManager module (for WLD deployment)
- DNS Server PowerShell module (for DNS zone creation, Windows only)
- Network: Trunk port group on physical vCenter for VLAN tagging
- OVA files: Nested ESXi 9.0.1 and VCF SDDC Manager 9.0.1.0
- Software depot (offline HTTP server or VMware online account)
- VyOS router configured with VLAN interfaces and BGP

## APIs Used

| API | Base URL | Used By |
|-----|----------|---------|
| VCF Installer | `https://<installer>/v1/` | Fleet deployment (tokens, bundles, depot, releases, bringup) |
| SDDC Manager | `https://<sddcm>/v1/` | WLD + Edge (tokens, clusters, hosts, network-pools, edge-clusters, tasks) |
| vCenter REST | `https://<vcenter>/api/` | Supervisor (session, cluster, storage, network, namespaces, content-library) |
| NSX Policy | `https://<nsx>/policy/api/v1/` | Supervisor (tier-0 lookup) |
| NSX Management | `https://<nsx>/api/v1/` | Supervisor (edge-cluster lookup) |
